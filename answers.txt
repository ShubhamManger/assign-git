• What does fast-forward in Git mean?
When merging one branch to another without any conflicts it is said to be fast forwarded.

• Why should you have different branches?
We should different branches so that an individual can work locally and after completing his/her work he/she can upload it to git so that other people on the project can review it and use it for development.
